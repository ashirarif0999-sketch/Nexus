# Performance Optimization Plan for EntrepreneursPage.tsx

## Current Performance Issues (Lighthouse Metrics)

Based on the provided Lighthouse report, the EntrepreneursPage has significant performance problems:

### Core Web Vitals
- **FCP**: 7.3s (Poor - should be <1.8s)
- **LCP**: 13.9s (Poor - should be <2.5s) 
- **TBT**: 9,860ms (Poor - should be <200ms)
- **Speed Index**: 17.8s (Very poor)
- **Overall Score**: 82/100

### Key Issues Identified

1. **Massive Icon Loading (Critical Path Blocker)**
   - Loading 1000+ individual Lucide React icons (~3-5KB each)
   - Network dependency tree shows 25.5s critical path latency
   - Icons like `user-round-plus`, `tram-front`, `text-select` etc. loading sequentially
   - **Impact**: Blocks LCP, causes long main-thread tasks

2. **Unoptimized Images**
   - Total image size: 10,219KB (mostly from Pexels)
   - Largest image: 2.6MB (should be <100KB)
   - No responsive images, no modern formats (WebP/AVIF)
   - **Impact**: Slow LCP, large network payload

3. **Render-Blocking Resources**
   - External CSS: `boxicons.min.css` (12KB), `inter.css` (2KB)
   - **Impact**: Delays initial render

4. **Forced Reflows**
   - JavaScript geometric property queries causing layout thrashing
   - Likely from Sticky component recalculations
   - **Impact**: Poor TBT, janky scrolling

5. **Large JavaScript Bundle**
   - Main chunk: 906KB (unminified)
   - Unused JavaScript: 505KB
   - Long main-thread tasks (20 found)
   - **Impact**: Slow parsing/execution

## Proposed Optimization Strategy

### Phase 1: Critical Path Optimization (High Impact)
1. **Icon System Overhaul**
   - Replace individual Lucide React icons with single icon font
   - Use `@lucide/icons` with tree-shaking
   - Lazy load icon components
   - **Target**: Reduce critical path from 25.5s to <5s

2. **Image Optimization Pipeline**
   - Implement responsive images with `srcset`
   - Convert to WebP/AVIF formats
   - Add lazy loading for below-fold images
   - Compress to <100KB per image
   - **Target**: Reduce image payload from 10MB to <500KB

### Phase 2: Bundle & Rendering Optimization (Medium Impact)
3. **JavaScript Bundle Splitting**
   - Code-split EntrepreneurCard components
   - Implement route-based splitting
   - Remove unused imports (Card, CardHeader, CardBody)
   - **Target**: Reduce main bundle by 50%

4. **Critical CSS Inlining**
   - Inline above-the-fold CSS
   - Defer non-critical external stylesheets
   - **Target**: Eliminate render-blocking CSS

### Phase 3: Runtime Performance (Low Impact)
5. **Layout Stability**
   - Optimize Sticky component (replace with CSS sticky if possible)
   - Minimize forced reflows from geometric queries
   - Implement virtual scrolling for large lists
   - **Target**: Reduce TBT to <200ms

6. **Memory Management**
   - Properly clean up timeouts/intervals
   - Implement proper component unmounting
   - **Target**: Eliminate memory leaks

## Expected Outcomes

- **FCP**: 7.3s → 1.5s (80% improvement)
- **LCP**: 13.9s → 2.0s (85% improvement)  
- **TBT**: 9,860ms → 150ms (98% improvement)

- **Total Bundle Size**: 20MB → 2MB (90% reduction)
